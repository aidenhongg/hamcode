# Core ML stack — tested on RTX 4090 (Ada, sm_89) Runpod.
#
# IMPORTANT: do NOT pip-install `torch` from this requirements file directly.
# Use scripts/run_runpod.sh which pins torch==2.6.0 from the cu124 index:
#
#   pip install --force-reinstall --index-url https://download.pytorch.org/whl/cu124 torch==2.6.0
#
# Why this exact version:
#   - PyPI default index serves a CPU-only torch which won't see your GPU.
#   - cu128 index serves +cu130 wheels at HEAD (need driver 12.8+, Runpod
#     4090 templates ship driver 12.4).
#   - transformers >= 4.51 added check_torch_load_is_safe() (CVE-2025-32434);
#     it refuses .bin loads on torch < 2.6 — and microsoft/longcoder-base is
#     .bin only (no safetensors mirror on HF). 2.6 is therefore the floor.
#   - torch 2.10+ caps setuptools<82, which breaks modern pip envs.
#   - torch 2.7+ bundles NCCL 2.21+ and references ncclCommWindowDeregister,
#     which can hit "undefined symbol" errors if a base image's older system
#     libnccl shadow-loads ahead of torch's bundled one. 2.6.0 bundles NCCL
#     2.20.x, sidestepping that whole class of issue.
#
# If the base image ships torchvision/torchaudio, uninstall them to avoid
# transformers' lazy-vision import exploding:
#   pip uninstall -y torchvision torchaudio
#
# We don't list torch here because pip-resolving against the default PyPI
# index would silently grab the CPU build. The bootstrap script does it.
